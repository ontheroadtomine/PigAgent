# PigAgent — Agent Loop 说明

> 本文档记录了 PigAgent（Codex 风格桌面软件 Agent）的核心运行逻辑：**Agent Loop**（智能体循环）。

## 什么是 Agent Loop？

Agent Loop 是 PigAgent 的核心工作模式。它遵循一个持续的 **Reason → Act → Observe** 循环，直到用户请求被真正完成。

```
┌─────────────────────────────────────┐
│         用户提出请求                   │
└──────────┬──────────────────────────┘
           ▼
┌─────────────────────────────────────┐
│  1. Reason（推理）                    │
│     - 理解用户意图                     │
│     - 拆解任务步骤                     │
│     - 决定下一步需要什么信息或操作       │
└──────────┬──────────────────────────┘
           ▼
┌─────────────────────────────────────┐
│  2. Act（行动）                       │
│     - 调用工具（Tool）                 │
│     - 执行命令、读取文件、搜索代码等     │
└──────────┬──────────────────────────┘
           ▼
┌─────────────────────────────────────┐
│  3. Observe（观察）                   │
│     - 查看工具返回的结果                │
│     - 分析输出、错误、状态              │
│     - 判断是否完成                     │
└──────────┬──────────────────────────┘
           │
    ┌──────┴──────┐
    ▼              ▼
  完成            未完成
    │              │
    │      回到 Reason
    │      继续循环
    ▼
┌─────────────────────────────────────┐
│         输出最终答案                   │
└─────────────────────────────────────┘
```

## 详细说明

### 1. Reason（推理）

- 分析用户当前请求的意图。
- 拆解为可执行的子任务。
- 决定下一步需要调用哪个工具、读取哪个文件、或执行什么命令。
- 如果上一步工具调用失败，分析失败原因并调整策略。

### 2. Act（行动）

调用可用的工具来获取信息或修改工作空间。主要工具包括：

| 工具 | 用途 |
|------|------|
| `workspace_list` / `workspace_files` | 浏览工作空间文件结构 |
| `workspace_search` | 搜索代码中的符号、路由、关键字 |
| `file_read` / `file_read_many` | 读取文件内容 |
| `file_write` | 创建或覆盖文件 |
| `apply_patch` | 应用补丁修改文件 |
| `shell_exec` | 执行 shell 命令（测试、构建、运行） |
| `web_fetch` | 获取公开 URL 内容 |
| `weather_current` | 获取实时天气 |

### 3. Observe（观察）

- 检查工具返回的结果（成功 / 失败 / 输出内容）。
- 判断当前任务是否完成。
- 如果完成，输出最终答案；否则回到 Reason 继续循环。

---

## 核心原则

1. **先看再改** — 修改代码前优先阅读工作空间和现有文件。
2. **批量读取** — 需要分析多个文件时，使用 `file_read_many` 一次读取，避免逐个读取浪费轮次。
3. **聚焦命令** — 使用 `shell_exec` 执行测试、构建等操作时保持命令简洁。
4. **不暴露密钥** — 不在输出中泄露任何敏感信息。
5. **完成即止** — 用户请求真正完成后才输出最终答案，不拖泥带水。

## 示例流程

```
用户: "帮我看看项目里有哪些 Python 文件"

Agent Loop:
  Reason  → 需要查看工作空间文件
  Act     → workspace_list()
  Observe → 看到文件列表
  Reason  → 过滤出 .py 文件
  Act     → 无（已有信息）
  Observe → 可以回答
  输出最终答案
```
